# Security policy

## Supported versions

Security fixes are applied to the latest pre-1.0 release. After 1.0, the support policy will be documented per release line.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private security advisory form:

`Security` → `Advisories` → `Report a vulnerability`

Include the affected version, reproduction steps, impact and any suggested mitigation. Reports will be acknowledged as soon as practical.
